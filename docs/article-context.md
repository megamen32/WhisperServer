# Контекст для статьи

Этот файл — короткая справка для статьи про локальную диктовку и транскрибацию.

## Главный тезис

Писать руками не обязательно там, где можно говорить обычной речью и получать готовый текст. Локальный WhisperServer нужен не как игрушка, а как инфраструктурный слой: один endpoint для Mac/iPhone/Android/Telegram/скриптов, одинаковое качество, приватность и fallback.

## Что показать в статье

- Сервер совместим с OpenAI v1 Audio Transcriptions, поэтому к нему можно подключать существующие клиенты и SDK.
- Streaming нужен для UX: текст появляется сразу, а не после полной обработки.
- Web UI и Telegram bot закрывают ручные сценарии.
- Python client с fallback закрывает desktop/app сценарии: удалённый сервер недоступен — транскрибация всё равно работает локально.
- CUDA broker нужен, когда на одном GPU живут несколько проектов: ASR, vision, локальные LLM и т.п.
- Встроенная диктовка удобна, но хуже для длинной речи, смешанных языков и сценариев, где нельзя зависеть от внешнего облака.

## Мини-демо

```bash
curl -sS http://127.0.0.1:7653/v1/audio/transcriptions \
  -H "Authorization: Bearer dev-local-key" \
  -F "file=@voice.ogg" \
  -F "model=whisper-1" \
  -F "language=ru"
```

## Что не обещать

- Это не realtime ASR в смысле распознавания микрофонного потока по 20 мс.
- Качество зависит от модели, записи, языка и железа.
- Если секрет уже был в истории, его нужно считать скомпрометированным даже после чистки git.
